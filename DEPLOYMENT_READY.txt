================================================================================
                        QUICK TROUBLESHOOTING
================================================================================

"I deployed but site is down!"
  1. Check logs: tail -f storage/logs/laravel.log
  2. Verify database: php artisan tinker → DB::connection()->getPdo()
  3. Clear caches: php artisan cache:clear
  4. If still broken: bash rollback.sh
  5. Contact: (your team)

"Queries are slow!"
  1. Verify caches were created: php artisan config:cache
  2. Check if route:cache ran: php artisan route:cache
  3. View active queries: Enable query logging in AppServiceProvider

"CSV export failing!"
  1. Check rate limit: 10 exports per minute
  2. Wait 1 minute and try again
  3. Check logs for memory issues
  4. Verify disk space available

"Getting 500 errors!"
  1. Check logs: tail -f storage/logs/laravel.log
  2. Clear views: php artisan view:clear
  3. Restart PHP: sudo systemctl restart php8.2-fpm
  4. Restart web server: sudo systemctl restart nginx

"App seems to hang!"
  1. Check database connection (might be timeout)
  2. Verify web server isn't maxed out
  3. Check if cache is full
  4. Look for n+1 queries if using fresh code

For more help, see QUICK_REFERENCE.md - Troubleshooting section

================================================================================
                           ROLLBACK PROCEDURE
================================================================================

If deployment causes issues:

Step 1: Stop the bleeding
  bash rollback.sh
  ↳ Automatically reverts to previous version
  ↳ Takes 2-3 minutes
  ↳ No manual steps required

Step 2: Verify Success
  - Application loads: http://localhost (or your domain)
  - Check logs: tail -f storage/logs/laravel.log
  - No errors should appear

Step 3: Investigate Root Cause
  - Check what changed: git diff HEAD~1
  - Review deployment logs
  - Check .env configuration
  - Contact team members

Step 4: Fix and Re-deploy
  - Fix the issue
  - Commit changes: git commit -am "Fix deployment issue"
  - Re-deploy: bash deploy.sh

If rollback fails:
  Manual steps (see PRODUCTION_READINESS_DEPLOYMENT.md):
  - php artisan config:clear
  - php artisan route:clear
  - php artisan view:clear
  - php artisan cache:clear
  - git reset --hard HEAD~1
  - composer install --no-dev
  - npm run build
